作者： Lian

webpack

是一个现代JavaScript应用的静态模块打包器(module bundle)。当webpack处理应用程序时，它会用递归方式构建一个依赖关系图，涵盖应用程序需要的每一个模块，然后将其打包成一个或多个bundle。

使用webpack，你需要先理解4个核心概念。

entry,入口

指示webpack应该使用哪个模块，来作为构建其内部依赖图的开始。接着，webpack将依次找出入口模块依赖的其他模块或库。
接下来，我们来看一个entry配置的最简单例子。

module.exports = {
  entry:'./path/to/my/entry/file.js'
}

output，出口

告诉webpack在哪里输出它创建的bundles,以及如何命名这些文件。

const path = require('path')

module.exports = {
  entry:'./path/to/my/entry/files.js',
  output:{
    path:path.resolve(__dirname,'dist'),
    filename:'my_first_webpack.bundle.js'
  }
}

在上面的例子中，我们通过output.path和output.filename，来告诉webpack 生成bundle的名称，以及我们想要生成(emit)到哪里。

loader,加载器

wepack自身只理解javascript，如果我们希望处理一些非JavaScript文件，怎么办？
对于这个问题，webpack提供了loader机制，它可以将其他类型的文件转化为webpack能够处理的有效模块。
在webpack配置中，loader有两个目标。
1、使用test属性，识别出与loader相对应的那些文件。
2、使用use属性，转换这些文件，添加至依赖图，最终添加到bundle中。

const path = require('path')

const config = {
  entry:'',
  output:{
    path:'',
    filename:''
  },
  module:{
    rules:[
      {test:/\.txt$/,use:'raw-loader'}
    ]
  }
};

module.export = config

plugins,插件

相比于loader专注于转换模块类型的单一功能，插件用于执行各种特定的任务。它的适用范围：从打包优化和压缩，一直到重新定义环境中的变量。

使用插件，需要两个步骤：
1、使用require()引入，添加到plugins数组中，
2、使用new操作符来创建它的一个实例。

const HtmlWebpackPlugin = require('html-webpack-plugin'); // 通过 npm 安装
const webpack = require('webpack'); // 用于访问内置插件
const path = require('path');

const config = {
  entry: './path/to/my/entry/file.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'my-first-webpack.bundle.js'
  },
  module: {
    rules: [
      { test: /\.txt$/, use: 'raw-loader' }
    ]
  },
  plugins: [
    new webpack.optimize.UglifyJsPlugin(),
    new HtmlWebpackPlugin({template: './src/index.html'})
  ]
};

module.exports = config;
