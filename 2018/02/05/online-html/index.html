HTML

英文全称 Hypertext Markup Language， 超文本标记语言，用于创建web网页。

W3C

英文全称 World Wild Web consortium，一个国际委员会组织。它是专门为定义HTML和XHTML标准而成立的。

HTML哲学

我们应该把精力集中在文档的内容上。那些看上去很酷的页面只不过是全景中的一个亮点而已,有内涵的内容才会吸引越来越多的读者。
委员会在设计HTML标准之初，达成了这样一个约定：内容高于视觉表现
。所以，我们在使用html时，需要按照他的设计方式来使用：表明文档的结构，以便浏览器能够正确显示它们的内容。

我们来看一下，最简单的HTML文档




  
    
    hello,world

你可以随心所欲将喜欢的任何东西加入HTML代码中，可能有些令人惊奇，但事实却是HTML文档仅仅由3种不同的元素组成：标签，注释，文本。

注释

注释，拆分为”注”与”释”两个字，分别代表标注和解释。我们在编程时，往往会在某段代码附近，写上标注，用于解释这段代码的意图，这就是我们的注释。
我们来看看html中注释的语法规则：
注释以<!--开头，以-->结束，中间的文本则是我们的注释内容。注释的内容不会在浏览器中显示出来，但是我们可以在源代码中查看。

我们来演示一下

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <!-- 此处是我们的网页标题 -->
    <title>hello,world</title>
  </head>
  <body>

  </body>
</html>

和其他编程语言一样，如果没有注释，一个充满标签的原始HTML文档，很快就难以阅读。因此，强烈建议大家在编程时，培养注释的好习惯，保持清晰的编程思路。

html元素(elements)

html文档由元素组合而成。什么是html元素？我们来看个示例

My cat is very grumpy

我们的HTML元素有3个组成部分：开始标签，结束标签，内容。
1、开始标签：以<开头，以>结束，标签中的第一个单词，是标签的名称，描述它的功能。接下来的其他单词，是标签的属性，用于描述标签的行为。
2、结束标签：以</开头，以>结束。大多数标签是成对存在的，良好的编程习惯，是不能忘了写结束标签。如：省略

元素的结束标签，就是很多前端程序员容易犯的错误。
3、内容：包含在开始标签和结束标签中的内容，很多时候是文本。
示例：

<p class="main">My cat </p>

嵌套元素
你可以把元素放在其他元素之中，我们称之为嵌套。我们一般称嵌套在外面的元素称为父元素，里面的元素称为子元素。注意，子元素的开始标签和结束标签都被包含在父元素里头。
我们来看个例子

<p>My cat is <strong>very</strong> grumpy.</p>

在这个例子中，p元素嵌套了strong元素，表示对very这个单词进行强调。

HTML元素有2两种重要的分类：
1、块级元素：该元素在页面中以块状方式呈现，即相对于前面的内容，它会另起一行，其后的内容也会被挤到下一行展现。常见的块级元素有：div,p
2、内联元素：通常出现在块级元素内部（嵌套），并包裹文档内容的小部分。它一个显著特性就是元素不换行。常见的内联元素有：a，span

<p>My cat is <span>very</span> grumpy.</p>

HTML结构
HTML的整体结构是非常简单清晰的。首先是HTML元素，它包裹了整个完整的网页，故HTML元素又被称为根元素。
head元素，网页的头部，主要包含一些网站属性信息，如：标题，字符集，css样式等。
body元素，网页的主体，你访问网页时，看到的所有内容，包括文本，图片，音频等内容，都是在主体定义的。

html->(head+body)

head元素

head元素包含一些页面的元数据，用于定义网页的相关属性。这些内容不会在浏览器中显示。
示例：

<head>
  <meta charset="utf-8">
  <title>My test page</title>
</head>

接下来，我们来看看head元素常见的子元素。

1、title元素，用于为网页添加标题。

<head>
  <meta charset="utf-8">
  <!-- 此处添加标题 -->
  <title>My test page</title>
</head>

2、meta元素，用于定义网页属性的元数据。元数据的官方定义有点拗口，描述数据的数据。它含有页面的相关信息，包括样式、脚本及数据，能帮助一些软件 (如搜索引擎， 浏览器等等）更好地运用和渲染页面。
常见的开发技巧，把你网站的关键字放在keyword和description元数据中，可以帮助搜索引擎更好找到你的网站。

3、link元素，用于引入外部的文件。常见的有：引入外部的css文件。

<link rel="stylesheet" href="my-css-file.css">

此外，它还可以帮助你实现网站添加小图标的功能。

语义化原则

语义，又称顾名思义。我们看到什么，就会知道它代表什么。举个例子, 我们知道红色交通灯表示“停止”，绿色交通灯表示”通行“。我们希望生活中事情井井有序，这样我们凭借着以前的经验或常识，就能做出正确的判断。
同样的道理，在编程时，我们要遵守语义化原则。我们需要确保使用正确的元素来给予内容正确的意思、作用以及外形。如：使用

元素来定义您的页面上用来表示顶级标题的文本，而不是其他的元素。

body元素

body元素是网页的主体内容。大部分的网页的主要工作是编辑文本内容，以便浏览器能正确显示。
大部分的文本结构由标题和段落组成。 不管是小说、报刊、教科书还是杂志等。

标题(heading)元素
用于描述标题元素，包含六个不同的级别，h1到h6，其中h1级别最高，h6级别最低。

p元素
表示文本的一个段落。该元素是块级元素，通常表现为一整块与相邻文本分离的文本。以前的程序员喜欢省略p元素的结束标签，虽然浏览器不会报错，但这是不好的习惯，这里特别提醒。

div元素
这是开发中使用最多的元素，在语义上不代表任何特定类型的内容。它一般结合class和id属性，对相关元素进行分组，以便更好进行样式处理。

span元素
常用的行内元素，一般与p元素嵌套使用。

em元素
用于需要加粗的文本内容，表示强调。

strong元素
表示文本十分重要，一般用粗体。与em同时表示强调，但strong强调意味更加重。

pre元素
表示预定义格式文本，这些文本通常按照原文件中的编排，以等宽字体的形式展现出来。
比如：文本块中有需要行或者垂直对齐的文本,你希望这些内容即使在浏览器窗口大小改变时也不会发生变化，这种情况可以采用pre标签。
示例：

<pre><span class="line"><span class="tag">&lt;<span class="name">div</span> <span class="attr">id</span>=<span class="string">"app"</span>&gt;</span></span><br><span class="line">  {{ message }}</span><br><span class="line"><span class="tag">&lt;/<span class="name">div</span>&gt;</span></span><br></pre>

ul，英文全称为unordered list，无序列表，表示没有特定顺序的一种列表(如洗衣店列表或者杂货店列表)。它们用 和 标签来定义边界，列表数据项item用

    ol元素
    ol,英文全称ordered list，有序列表，表示带数字编号的列表。有序列表用

      标签来限定边界，格式与无序列表相同。
      常用属性：
      1、reversed
      reverse，中文是翻转，默认情况，列表是顺序排序。不过，如果你需要逆序，可通过配置reversed属性来实现。它是布尔属性。
      2、start
      整数属性，规定列表序号的开始值。
      3、type

    form元素
    表示了文档中的一个区域，这个区域包含有交互控制元件，如文本输入框，单选按键，多选按键，下拉菜单等，用于获取用户的输入数据。
    常见属性：
    1、action
    规定接收和处理表单数据的应用程序的URL。该值有可能被button和input的formation属性覆盖。

    2、enctype
    浏览器在把数据传给服务器之前，会专门对表单的数据进行编码，这样在传输过程中数据才不会被打乱或破坏。
    默认格式是application/x-www-form-urlencoded。
    当传送的表单数据包括文件时，一般采用多部分表单格式 multipart/form-data。
    当无法访问表单处理服务器，需要将表单数据通过邮件发送时，才会采用text/plain格式。
    该值有可能会被button和input的formenctype属性覆盖。

    3、accept-charset
    指定字符集，服务器必须支持该字符集，才能正确解释表单中的数据。
    可不填，默认值为unknown，意味着表单的字符集与该HTML的字符集相同。

    4、method
    必填，有两种方法：post get
    如果采用Post方法，浏览器将会安装下面两步来发送数据。首先，浏览器与服务器建立联系，一旦建立联系成功，浏览器按分段传输的方法将数据发送给服务器。
    如果采用get方法，首先，浏览器与服务器建立联系，一旦建立联系，浏览器会将所有数据附在表单的url之后，用问号隔开，一次性发送给服务器。

    5、target
    用来指示在提交表单之后，在哪里显示收到的回复。
    _self: 在当前HTML4或HTML5文档页面重新加载返回值。这个是默认值。
    _blank: 以新的HTML4或HTML5文档窗口加载返回值。

    <!-- 一个简单的表单，这个表单会发送一个 GET 请求 -->
<form action="">
  <label for="GET-name">Name:</label>
  <input id="GET-name" type="text" name="name">
  <input type="submit" value="Save">
</form>

    4、file，可以让用户选择文件。一般结合accept属性，定义文件类型。

    7、submit，表单提交的按键。

    checked

    name
    控件名称，与表单数据一起提交

    URL
    在互联网上有数以百万甚至数以亿计的文档和资源,这些文档和资源在互联网上的惟一地址,这个地址被称为统一资源定位符( uniform resource locator, URL)。

    我们来看看URL的语法规则：
    protocol://server_domain_name/pathnαme/filename
    阅读URL，需要从后往前看。
    首先是filename，表示文档名称；
    其次是pathname，表示文件路径（又称目录结构）；
    然后是server_domain_name,表示服务器的互联网域名；
    最后是protocol，表示浏览器与服务器的通信协议。

    绝对和相对URL
    绝对URL是资源的完整地址，包含系统在web上寻找文档和它所在服务器所需的所有信息。
    相对URL只提供简写的文档地址，系统会自动将其与基本地址base address 合并起来，生成一个完整的文档地址。
    开发中推荐使用相对URL，想想看，当你需要将项目从一个目录前移到另一个目录，或一个服务器迁移到另一台服务器，使用相对URL则无需改动。使用绝对URL则需要修改所有URL，将是一件多么困难的事情。

    http端口
    端口是客户端与服务器连接的通信端口号码。服务器处理可以提供web文档服务外，还可以执行其他功能，如：电子邮件，FTP文件读写等，按照服务类型不同，连接到服务器的不同端口。
    web服务器的默认端口是80，而使用HTTPS的服务器使用端口443.
